## Phase 1 — Security Hardening

> **Goal**: Fix all security issues identified in the codebase audit. These are required before any real patient data enters the system.

- [ ] **1.1** Add CSRF protection — install Flask-WTF, call `CSRFProtect(app)` in `app/__init__.py`, add `{{ csrf_token() }}` hidden field to every `<form>` in templates that uses `POST`. Verify all AJAX POST calls include the CSRF token header.
  - Files: `app/__init__.py`, `requirements.txt`, every template with `<form method="POST">`
  - Test: confirm all POST routes reject requests without CSRF token; all forms still work with it

- [ ] **1.2** Add PIN brute-force rate limiting — in the `verify_pin` endpoint in `routes/auth.py`, add a failed-attempt counter per user session. After 5 consecutive wrong PINs, lock the user out for 5 minutes. Store attempt count in the session or a lightweight in-memory dict with expiry.
  - Files: `routes/auth.py`
  - Test: verify 6th wrong PIN returns 429; correct PIN after lockout expires works

- [ ] **1.3** Migrate secrets out of `config.py` — replace all hardcoded credentials with `os.getenv()` calls with empty-string defaults. Create `config.example.py` as a template showing which env vars to set. Add `SMTP_SERVER`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM` placeholder entries (already referenced by `agent_service.py` but not in config).
  - Secrets to migrate: `SECRET_KEY`, `AC_LOGIN_PASSWORD`, `PUSHOVER_USER_KEY`, `PUSHOVER_API_TOKEN`, `OPENFDA_API_KEY`, `LOINC_USERNAME`, `LOINC_PASSWORD`, `UMLS_API_KEY`, `VIIS_PASSWORD`, `PDMP_PASSWORD`
  - Files: `config.py`, CREATE `config.example.py`
  - Keep existing hardcoded values as fallback defaults during development only — production guide should instruct setting env vars

- [ ] **1.4** Require password confirmation for sensitive admin actions — when an admin changes another user's username or role via `/admin/users`, require the admin's own password in the form. Add an audit log entry for the change.
  - Files: `routes/admin.py`, `templates/admin_users.html`

## Phase 2 — Bug Fixes

> **Goal**: Fix all known bugs identified in the audit.

- [ ] **2.1** Fix TCM deadline logic in `app/services/billing_rules.py` — the `_add_business_days()` call in TCM detection should compare the calculated contact deadline against `datetime.now()` to determine if the window has already passed. If today is past the 2-business-day contact deadline, mark the opportunity as `overdue` rather than showing it as a future action.
  - Files: `app/services/billing_rules.py`

- [ ] **2.2** Fix drug name matching in `routes/intelligence.py` `lab_interpretation()` (~L444) — currently does `med.drug_name.split()[0].lower()` which misses combination drugs (e.g., "lisinopril-HCTZ"). Change to match against the full drug name AND the first word, and also match on `med.rxnorm_cui` when available (RxCUI matching is more reliable than text).
  - Files: `routes/intelligence.py` (lab_interpretation function)

- [ ] **2.3** Fix formulary gap keyword matching in `routes/intelligence.py` `formulary_gaps()` (~L700) — add common abbreviations to the drug keyword lists (e.g., "HCTZ" for hydrochlorothiazide, "ASA" for aspirin, "MVI" for multivitamin). Also match on RxCUI → RxClass mapping when structured data is available.
  - Files: `routes/intelligence.py` (formulary_gaps function)

- [ ] **2.4** Fix scheduled deactivation timezone in `routes/auth.py` (~L535) — when parsing `deactivation_date` from the form, apply the server's local timezone (or user-configured timezone) before converting to UTC. Currently `.replace(tzinfo=timezone.utc)` treats the naive datetime as UTC even though the user entered it in local time.
  - Files: `routes/auth.py`

- [ ] **2.5** Fix spell check silent API failure in `app/services/clinical_spell_check.py` `_enrich_with_api()` — when the API call raises an exception, append a result entry with `confidence=0` and `suggestion="API unavailable — check connection"` so the user knows enrichment was attempted but failed, rather than silently returning nothing.
  - Files: `app/services/clinical_spell_check.py`

- [ ] **2.6** Fix Tesseract path portability in `config.py` — dynamically detect Tesseract from `PATH` using `shutil.which('tesseract')`, falling back to the bundled `tesseract/tesseract.exe` relative path, then the hardcoded path as last resort. This makes deployment portable across machines.
  - Files: `config.py`

- [ ] **2.7** Fix delayed message delivery TODO in `agent_service.py` (L419) — the `job_delayed_message_sender` has a `# TODO: integrate with NetPractice scraper for actual delivery` comment. Determine if delayed messages should be delivered via Pushover (already available) or if NetPractice messaging is the intended channel. If Pushover, wire the `send_notification()` call. If NetPractice, document the dependency and leave the TODO with a clear blocker note.
  - Files: `agent_service.py`

**Phase 2 test gate**: `python test.py` passes. Verify TCM deadline with a past-discharge-date test case.

## Phase 3 — Intelligence Layer Completion

> **Goal**: Wire up proactive intelligence features. The API endpoints exist (lab_interpretation, drug_safety, guidelines, formulary_gaps, patient_education) but lack proactive background jobs and UI integration in the patient chart.

- [ ] **3.1** Create NLM Conditions API client — `app/services/api/nlm_conditions.py` following the `BaseAPIClient` pattern. Endpoint: `https://clinicaltables.nlm.nih.gov/api/conditions/v3/search`. Methods: `search_conditions(query)` → returns related condition names and codes. Wire `_save_to_structured_cache()` to store results.
  - Files: CREATE `app/services/api/nlm_conditions.py`

- [ ] **3.2** Create `nlm_conditions_cache` model — add `NlmConditionsCache` to `models/api_cache.py` (search_term PK, conditions JSON, cached_at, ttl_days=90). Import in `models/__init__.py`. Create `migrate_add_nlm_conditions_cache.py`.
  - Files: `models/api_cache.py`, `models/__init__.py`, CREATE `migrate_add_nlm_conditions_cache.py`

- [ ] **3.3** Add Drug Recall background scan — create a scheduled job in `agent/scheduler.py` that runs daily at 3 AM. It queries all distinct `rxnorm_cui` values from `patient_medications` (active status), checks each against the existing `OpenFDARecallsService.check_drug_list_for_recalls()`, and creates `Notification` records for the user if any active recalls are found. Include recall count in the morning briefing data.
  - Files: `agent/scheduler.py`, `agent_service.py`, `routes/intelligence.py` (morning_briefing function — add recall summary)

- [ ] **3.4** Add pre-visit billing opportunity calculation — create a scheduled job in `agent/scheduler.py` that runs nightly at 8 PM. For each patient on tomorrow's schedule, run `billing_rules.evaluate_patient(mrn)` and store results as `BillingOpportunity` records. Display billing opportunity badges on the dashboard schedule cards.
  - Files: `agent/scheduler.py`, `agent_service.py`, `routes/dashboard.py` (add opportunity count to schedule data), `templates/dashboard.html` (add badge)

- [ ] **3.5** Integrate intelligence panels into patient chart — add collapsible "Intelligence" section to `templates/patient_detail.html` (or patient chart widget area) with lazy-loaded panels:
  - Drug Safety (calls `/api/patient/<mrn>/drug-safety`)
  - Lab Interpretation (calls `/api/patient/<mrn>/lab-interpretation`)
  - Guidelines (calls `/api/patient/<mrn>/guidelines`)
  - Formulary Gaps (calls `/api/patient/<mrn>/formulary-gaps`)
  - Patient Education (calls `/api/patient/<mrn>/education`)
  - Each panel loads on-demand (click to expand) via fetch() to avoid slowing chart load.
  - Files: `templates/patient_detail.html` (or `patient_chart.html`), add JS fetch handlers

- [ ] **3.6** Add patient education → delayed message draft integration — when a user clicks "Send to Patient" on a patient education result, auto-create a `DelayedMessage` draft with the MedlinePlus content as the message body, pre-filled recipient, and status `pending`. Redirect to `/messages` for review.
  - Files: `routes/intelligence.py` (add POST endpoint), `templates/patient_detail.html` (add button)
